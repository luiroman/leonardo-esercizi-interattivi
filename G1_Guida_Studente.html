2. Fondamenti TCP/IP

Perché TCP/IP è fondamentale per la cybersecurity? La rete è il campo di battaglia della cybersecurity. I tool di sicurezza (firewall, IDS, SIEM) sono essenzialmente "lettori di pacchetti" — senza capire come è fatto un pacchetto, non si capiscono i tool. Ogni concetto di networking ha un'applicazione diretta sia nella difesa che nell'attacco.

2.1 Il modello TCP/IP a 4 livelli

Il modello TCP/IP organizza la comunicazione di rete in 4 livelli, ciascuno con un ruolo specifico:

LivelloNomeRuoloProtocolliTool di sicurezza
4ApplicazioneInterfaccia con l'utenteHTTP, DNS, SMTP, SSHWAF, Proxy
3TrasportoComunicazione end-to-endTCP, UDPFirewall L4
2InternetIndirizzamento e routingIP, ICMP, ARPFirewall L3, IDS/IPS
1Accesso alla reteTrasmissione fisicaEthernet, Wi-FiPort Security, 802.1X

Ogni livello incapsula i dati del livello superiore aggiungendo il proprio header. Il processo inverso (de-incapsulamento) avviene sul dispositivo ricevente. La terminologia cambia per livello: messaggio (applicazione), segmento/datagramma (trasporto), pacchetto (internet), frame (accesso alla rete).

Nota sull'ambiguità OSI/TCP/IP. In ambito professionale si usa spesso la numerazione OSI (7 livelli) anche quando si parla di TCP/IP (4 livelli). "Layer 7" si riferisce all'applicazione, "Layer 3" all'IP, ecc. Non è tecnicamente preciso ma è una convenzione molto diffusa.

2.2 Layer 1: Accesso alla rete (Fisico + Data Link)

Indirizzo MAC

L'indirizzo MAC (Media Access Control) è un identificatore univoco a 48 bit assegnato dal produttore della scheda di rete. I primi 24 bit sono l'OUI (Organizationally Unique Identifier) che identifica il vendor. Formato: sei gruppi di due cifre esadecimali separati da : o - (es: AA:BB:CC:DD:EE:FF).

Frame Ethernet

Il frame Ethernet è l'unità di dati a livello 2. La struttura: Preamble (8 byte) + MAC destinazione (6 byte) + MAC sorgente (6 byte) + EtherType (2 byte — es: 0x0800 = IPv4) + Payload (46-1500 byte) + FCS (4 byte, controllo errori).

ARP (Address Resolution Protocol)

ARP risolve la corrispondenza IP → MAC nella rete locale. Funziona con request (broadcast: "chi ha questo IP?") e reply (unicast: "sono io, ecco il mio MAC"). La tabella ARP è una cache locale consultabile con il comando arp -a.

Switch vs Hub

Uno switch instrada i frame solo alla porta corretta usando la tabella CAM (Content Addressable Memory) che mappa MAC address → porta. Un hub replica tutto su tutte le porte.

Attacchi a Layer 2. L'attacco ARP Spoofing invia risposte ARP false per dirottare il traffico (tool: arpspoof, ettercap, Bettercap). Il MAC Flooding satura la tabella CAM dello switch costringendolo a comportarsi come un hub (tool: macof). Difese: Port Security, DHCP Snooping + DAI, 802.1X.

2.3 Layer 2: Internet (IP)

Il protocollo IP (RFC 791) gestisce l'indirizzamento logico e il routing dei pacchetti.

Header IPv4 — campi principali

CampoBitFunzione
Version4Sempre 4 per IPv4
IHL4Lunghezza header in word da 32 bit (min 5 = 20 byte)
Total Length16Dimensione totale del pacchetto
TTL8Time To Live — decrementato ad ogni hop

IP Spoofing. Il campo Source Address non è autenticato — chiunque può inserire un IP falso. Questo è alla base degli attacchi DDoS con amplificazione (DNS amplification, NTP amplification): l'attaccante manda richieste con l'IP della vittima come sorgente. Difesa: ingress filtering (BCP 38/RFC 2827).

2.4 Layer 3: Trasporto (TCP e UDP)

TCP (RFC 793)

TCP è connection-oriented e reliable: stabilisce una connessione prima di trasmettere, numera ogni segmento (Sequence Number), conferma la ricezione (ACK), ritrasmette i segmenti persi, e riordina quelli arrivati fuori sequenza.

TCP Flags — fondamentali per capire sia il funzionamento normale sia gli attacchi:

  • SYN: inizio connessione
  • ACK: conferma ricezione
  • FIN: chiusura ordinata
  • RST: chiusura immediata (reset)
  • PSH: push immediato dei dati all'applicazione
  • URG: dati urgenti

TCP Three-Way Handshake

Il meccanismo che stabilisce una connessione TCP:

  1. SYN: il client sceglie un ISN (Initial Sequence Number) pseudo-casuale e lo invia. Flag SYN attivo.
  2. SYN-ACK: il server sceglie il proprio ISN, mette in Acknowledgment Number il valore ISN_client + 1. Flag SYN e ACK attivi.
  3. ACK: il client conferma con Acknowledgment Number = ISN_server + 1. Solo flag ACK attivo.
Perché ISN casuali? Se i numeri di sequenza fossero prevedibili, un attaccante potrebbe iniettare pacchetti in una connessione TCP esistente (TCP Sequence Prediction Attack). I sistemi moderni usano ISN pseudo-casuali per prevenire questo.

UDP (RFC 768)

UDP è connectionless, unreliable e unordered. Header minimale di soli 8 byte. Usato quando la velocità è più importante dell'affidabilità (DNS, streaming, VoIP).

Attacchi a Layer 3-4. SYN Flood: esaurisce la coda di connessioni pendenti (difesa: SYN cookies). UDP Flood: inonda la vittima di pacchetti UDP. Port Scanning con Nmap: SYN scan (-sS, stealth), Connect scan (-sT), UDP scan (-sU). Xmas scan (-sX): flag FIN+PSH+URG per eludere firewall che filtrano solo i SYN.

2.5 Porte e servizi

Le porte TCP/UDP (0-65535) identificano l'applicazione sull'host. Classificazione IANA:

  • 0-1023: Well-Known Ports — servizi standard, richiedono root su Linux
  • 1024-49151: Registered Ports — assegnate su richiesta
  • 49152-65535: Dynamic/Ephemeral — porte sorgente dei client

Porte critiche da conoscere per la cybersecurity:

2.6 Incapsulamento

L'incapsulamento è il principio per cui ogni livello aggiunge il proprio header ai dati ricevuti dal livello superiore. In Wireshark, la vista a pannelli riflette questa struttura: il pannello centrale mostra i layer espandibili, dal frame Ethernet al payload applicativo.

Esempio completo: cosa succede quando apri https://www.google.com
  1. Il browser risolve il nome DNS (query UDP porta 53)
  2. Il browser apre una connessione TCP (three-way handshake verso porta 443)
  3. TLS handshake (negoziazione chiavi di cifratura)
  4. HTTP GET request (cifrata dentro il tunnel TLS)
  5. Il server risponde con il contenuto della pagina
  6. Il browser chiude la connessione (FIN-ACK)

Questo percorso sarà sviluppato praticamente nel Giorno 2 con Wireshark.

3. Indirizzamento IPv4 e Subnetting

3.1 Indirizzamento IPv4

Un indirizzo IPv4 è un numero a 32 bit in "dotted decimal notation" (es: 192.168.1.100). Ogni ottetto è un byte (0-255). L'indirizzo è composto da Network Part + Host Part, separati dalla subnet mask.

La notazione CIDR (RFC 4632) indica quanti bit appartengono alla parte di rete: /24 = i primi 24 bit sono rete, gli ultimi 8 sono host.

Indirizzi riservati in ogni subnet

  • Primo indirizzo (bit host tutti a 0): indirizzo di rete — identifica la subnet
  • Ultimo indirizzo (bit host tutti a 1): indirizzo di broadcast — raggiunge tutti gli host
  • Formula host utilizzabili: 2(32−n) − 2, dove n è il prefisso CIDR

Indirizzi privati (RFC 1918)

  • 10.0.0.0/8 — Classe A privata
  • 172.16.0.0/12 — Classe B privata
  • 192.168.0.0/16 — Classe C privata

Non instradabili su Internet. Il NAT traduce privato → pubblico per l'accesso esterno.

3.2 Subnetting: il metodo

Il subnetting è una competenza meccanica che si automatizza con la pratica. Il metodo più pratico usa le potenze di 2: la dimensione del blocco è 2(32−prefix). Per /26: 26 = 64 → i blocchi partono da multipli di 64.

Metodo in 5 passi

  1. Quanti host servono? — identificare il requisito
  2. Quanti bit per gli host? — trovare n dove 2n − 2 ≥ host richiesti
  3. Maschera di sottorete — 32 − n = prefisso CIDR
  4. Intervallo e broadcast — network address, range, broadcast
  5. Verifica — controllare sovrapposizioni e spreco
Errori comuni. Dimenticare di sottrarre 2 dal totale (network + broadcast). Confondere /26 con "26 host" (sono 62). Non capire che il broadcast della subnet 1 e il network della subnet 2 sono consecutivi ma appartengono a subnet diverse.
Il "magic number". È 256 meno l'ultimo ottetto della maschera. Per 255.255.255.192 (/26): 256 − 192 = 64. Le sottoreti saltano di 64 in 64: .0, .64, .128, .192.

3.3 VLSM: sottoreti di dimensione diversa

VLSM (Variable Length Subnet Mask) permette di assegnare maschere diverse a sottoreti diverse nello stesso blocco. Regola fondamentale: assegnare sempre partendo dalla sottorete più grande, perché le sottoreti grandi hanno vincoli di allineamento più stringenti.

Processo VLSM

  1. Ordinare le sottoreti dalla più grande alla più piccola
  2. Per ogni sottorete, calcolare la maschera minima: 2n − 2 ≥ host richiesti
  3. Ogni sottorete parte dove finisce la precedente
  4. Verificare che non ci siano sovrapposizioni

3.4 Supernetting e route summarization

Il supernetting è il processo inverso del subnetting: dato un insieme di rotte contigue, trovare la rotta più specifica che le copre tutte. Il metodo: scrivere gli indirizzi in binario e contare i bit comuni da sinistra. La summarization funziona solo se le rotte sono contigue e allineate ai confini della maschera.

Attenzione all'allineamento. Una /22 copre 4 rotte /24, ma solo a partire da un multiplo di 4 nel terzo ottetto (0, 4, 8, 12...). Le rotte 10.0.1.0–10.0.4.0 non si aggregano in una /22 perché 1 non è multiplo di 4.

4. Segmentazione e Sicurezza di Rete

4.1 La rete piatta e il lateral movement

Una rete piatta (flat network) è una topologia in cui tutti i dispositivi sono nello stesso dominio di broadcast e nella stessa subnet. Non ci sono barriere: qualsiasi host può comunicare direttamente con qualsiasi altro.

Blast radius. In una rete piatta, un singolo dispositivo compromesso espone l'intera infrastruttura. Il "blast radius" (raggio dell'esplosione) misura l'impatto di una compromissione — quanti sistemi sono esposti. La segmentazione riduce il blast radius.

Il lateral movement è la fase della cyber kill chain in cui un attaccante, dopo l'accesso iniziale a un host, si muove all'interno della rete per raggiungere asset di maggior valore. Le 4 fasi tipiche:

  1. Accesso iniziale — phishing, credenziali rubate, vulnerabilità
  2. Ricognizione interna — scansione rete (arp-scan, nmap)
  3. Movimento laterale — pass-the-hash, exploit SMB, pivot
  4. Escalation e obiettivo — domain admin, esfiltrazione, ransomware
Tecniche di lateral movement. (1) Pass-the-hash: uso di hash NTLM per autenticarsi senza password. (2) Exploit di servizi: vulnerabilità come EternalBlue (SMB). (3) Living off the land: strumenti legittimi (PowerShell, WMI, PsExec). La segmentazione interviene tra il passo 2 e il 3: se i server sono in un'altra VLAN con firewall, l'attaccante non li raggiunge.

4.2 Principi di segmentazione

  1. Separazione per funzione — server, utenti, ospiti, management, IoT in segmenti distinti
  2. Least privilege di rete — ogni segmento accede solo a ciò che gli serve. Default: deny all.
  3. Punti di controllo — firewall L3/L7 tra segmenti. Ogni flusso inter-VLAN passa per una regola.
  4. Monitoraggio ai confini — IDS/IPS e log tra segmenti. Il traffico Est-Ovest (laterale) diventa visibile.
Traffico Nord-Sud vs Est-Ovest. Nord-Sud è il traffico da/verso internet. Est-Ovest è il traffico laterale tra host interni. Nelle reti moderne, la maggior parte degli attacchi si sviluppa Est-Ovest — ed è invisibile senza segmentazione.

4.3 VLAN (IEEE 802.1Q)

Una VLAN (Virtual LAN) è una rete logica indipendente dal cablaggio fisico. Lo switch aggiunge un tag di 4 byte al frame Ethernet che identifica la VLAN di appartenenza.

  • Access port: membro di una sola VLAN, tag rimosso verso l'host
  • Trunk port: trasporta frame di più VLAN con i tag, usata tra switch e tra switch-router
Le VLAN da sole NON sono sicurezza. Le VLAN separano i domini di broadcast a layer 2, ma il routing inter-VLAN (layer 3) riconnette tutto. Senza regole firewall/ACL, la VLAN è solo organizzazione. VLAN + Firewall = Segmentazione vera.

Il routing inter-VLAN può avvenire tramite: (1) Router-on-a-stick con sub-interface, (2) Switch layer 3 con SVI, (3) Firewall come default gateway — la soluzione più sicura.

4.4 DMZ e zone di sicurezza

La DMZ (DeMilitarized Zone) è un segmento per servizi accessibili dall'esterno (web server, mail server), isolato sia dalla rete interna sia da internet. L'architettura classica prevede un firewall con 3+ interfacce o due firewall in serie.

Regole tipiche tra le zone:

  • Outside → DMZ: solo porte specifiche (HTTP/443, SMTP/25)
  • DMZ → Inside: molto restrittivo (es: solo il web server accede al DB su una porta specifica)
  • Inside → DMZ: consentito per management
  • DMZ → Outside: limitato (aggiornamenti, DNS)
Evoluzione: microsegmentazione. L'architettura Zero Trust (NIST SP 800-207) porta la segmentazione all'estremo: ogni workload ha il suo set di regole, indipendentemente dalla rete fisica. Tecnologie: VMware NSX, Cisco ACI, cloud security groups. Ne parleremo nel Modulo 3.

7. Strumenti da installare

Wireshark (obbligatorio — serve per il Giorno 2)

  1. Scarica da wireshark.org/download.html
  2. Installa:
    • Windows: esegui il .exe, accetta Npcap durante l'installazione
    • macOS: apri il .dmg, oppure brew install --cask wireshark
    • Linux: sudo apt install wireshark poi sudo usermod -aG wireshark $USER e rilogga
  3. Avvia Wireshark, seleziona l'interfaccia di rete attiva
  4. Avvia la cattura, naviga su 3-4 siti per 5 minuti, ferma e salva come cattura_[tuo_nome].pcapng

Problemi comuni: "Nessuna interfaccia" → verifica Npcap (Windows) o gruppo wireshark (Linux). "Traffico cifrato" → è normale! HTTPS cifra il payload, ma IP, DNS e handshake TLS restano visibili.

VirtualBox + Ubuntu Server (facoltativo ma consigliato)

  1. Scarica VirtualBox da virtualbox.org
  2. Scarica Ubuntu Server 24.04 LTS da ubuntu.com (~2.5 GB)
  3. Crea una VM: nome Ubuntu-Lab, 2 GB RAM, 20 GB disco, rete NAT
  4. Monta la ISO e avvia la VM
  5. Quando appare il menu iniziale, scegli "Try or Install Ubuntu Server"
  6. Segui il wizard interattivo passo per passo — lingua, tastiera, rete (lascia DHCP), disco (usa intero disco)
  7. Quando ti chiede nome, username e password: annotali! Ti serviranno per ogni accesso
  8. Alla domanda su OpenSSH: seleziona "Install OpenSSH server" — ci servirà per connetterci
  9. Completa l'installazione e riavvia
Attenzione: installazione interattiva, non automatica. Ubuntu Server può partire in modalità "autoinstall" senza chiedere le credenziali. Se succede, il sistema si installa senza utenti e non potrai accedere. Assicurati di seguire il wizard interattivo e di impostare username e password quando richiesto. Se ti ritrovi bloccato al login, reinstalla seguendo i passi sopra.

Accesso comodo via SSH (opzionale)

La console di VirtualBox è scomoda (no copia-incolla). Per lavorare meglio, configura l'accesso SSH:

  1. Nella VM, verifica SSH: sudo systemctl status ssh
  2. In VirtualBox: Impostazioni VM → Rete → Avanzate → Port Forwarding
  3. Aggiungi: Nome ssh, Protocollo TCP, Host Port 2222, Guest Port 22
  4. Dal tuo terminale: ssh -p 2222 tuoutente@127.0.0.1
